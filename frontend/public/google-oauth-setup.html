Fix Google redirect_uri_mismatch

This page reads your live Luna server config and shows exactly what to paste into Google Cloud Console.

Loading server config…

Step 1 — Click “Error details” on Google’s page

On the Google error screen, click If you are a developer of this app, see error details.

It shows the redirect_uri= Google actually received. That string must appear character-for-character in Google Console (see Step 2).

Step 2 — Edit the same OAuth client

  1. Open Google Cloud Console → Credentials
  2. Find OAuth 2.0 Client ID that exactly matches (loading…) above — not a different app name
  3. Application type must be Web application (not Desktop / Android / iOS)
  4. Authorized redirect URIs — add only: (loading…)
  5. Authorized JavaScript origins — add: https://luna.flowergrid.co.uk
  6. Remove wrong URIs (missing /api): …/auth/google/callback
  7. Click Save, wait 10 minutes

Step 3 — Still failing? Create a new Web OAuth client

  1. Credentials → Create credentials → OAuth client ID
  2. Application type: Web application
  3. Redirect URI: https://luna.flowergrid.co.uk/api/auth/google/callback
  4. Origin: https://luna.flowergrid.co.uk
  5. Copy the new Client ID + Secret into Coolify (replace old values)
  6. Coolify → Redeploy (restart container)

Step 4 — Coolify must match

GOOGLE_CLIENT_ID=<same as Step 2 or new client>
GOOGLE_CLIENT_SECRET=<secret for that client only>
GOOGLE_CALLBACK_URL=https://luna.flowergrid.co.uk/api/auth/google/callback
FRONTEND_URL=https://luna.flowergrid.co.uk
